This is a calibration lane, not a profit lane. The first 5-10 real trades exist to measure execution gap versus demo.

Lane Design

Dual Lane demo investigate stays on · real stays fail-closed

Upper EARN / INVESTIGATE tabs are dashboard view controls only. Real execution is handled by a separate lane state machine.

Prepared

  • Demo investigation is intended to remain always-on unless the global hard stop is used.
  • Real lane has separate states: `DISABLED`, `ARMED`, `LIVE`, `DRAINING`, `STOPPED`.
  • `Stop Real` means graceful real-only exit. It must not stop demo investigation.
  • Global `Stop` / kill switch remains the hard stop for everything.
  • Real lane is still calibration-only: manual unlock, one trade at a time, no auto-open.

First Real Rules

  • Use separate browser profiles for demo and real. Do not share one Libertex session.
  • First 5-10 real trades are execution-gap probes, not “earn mode” automation.
  • Each probe must log real spread, slippage, fill latency, and realized cost/R.
  • No automatic opportunistic real entry until calibration probes are complete.
  • Keep one concurrent real trade at most during calibration.

Stops

Suggested structure: global hard stop for all robot activity, plus a separate real-lane stop that bans new real entries and drains existing real positions. For the first $100 lane, the stop logic should be tighter than demo and should fail closed on any ambiguity.

  • Real lane stop: new real entries off, open real positions close out, demo continues.
  • Hard stop: scanner off, demo off, real off, global safety first.
  • Calibration phase: one trade at a time, no automatic re-entry after a real stop.
